# Design System Checklist App

## Overview

This project demonstrates a scalable design system built with React, TypeScript, and Storybook. The goal was to apply design system thinking by creating reusable UI components, extracting design tokens from a Figma design, and maintaining a consistent and user-friendly interface.

## Presentation Summary

> I created a scalable design system using React, TypeScript, and Storybook. I extracted design tokens for colors, typography, spacing, and border radius. I built reusable components such as Button, Input, Checkbox, and Card. The project supports automatic dark mode and uses Storybook to document components.

## Technologies Used

- React
- TypeScript
- Vite
- Storybook
- CSS Variables (Design Tokens)
- ESLint
- Prettier

## Design Tokens

The project uses CSS variables to define design tokens for:

- Colors
- Typography
- Spacing
- Border Radius

These tokens ensure visual consistency and make the system easy to maintain and scale.

## Reusable Components

The following presentational components were created:

- Button
- Input
- Checkbox
- Card

These components are reusable and contain no business logic, making them suitable for use across multiple features.

## Dark Mode

The project supports automatic system dark mode using the `prefers-color-scheme` CSS media query.

## Storybook

Storybook is used to document and test components in isolation. A story is included for the Button component.
